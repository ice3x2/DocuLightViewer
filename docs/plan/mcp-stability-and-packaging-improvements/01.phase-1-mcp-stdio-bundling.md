# Phase 1: MCP Stdio 번들링 + 의존성 정리

**해결 이슈**: #1 (asarUnpack 크기 폭증), #2 (zod 의존성 누락), #14 (shebang 누락)
**ADR 참조**: [ADR-001](./00-2.tech-decisions.md#adr-001-mcp-stdio-서버-esbuild-번들링)
**선행 조건**: 없음 (첫 번째 Phase)
**병렬 가능**: Phase 2, 3, 4와 독립

---

## 목표

패키징된 앱에서 MCP stdio 서버가 안정적으로 실행되도록 esbuild 번들링을 도입하고, `node_modules/**` asarUnpack을 제거하여 인스톨러 크기를 정상화한다.

---

## 아키텍처 참조

변경 전:
```
app.asar.unpacked/
├── src/main/mcp-server.mjs
├── src/main/frontmatter.js
└── node_modules/**              ← 수백MB 중복
```

변경 후:
```
app.asar.unpacked/
└── src/main/mcp-server.bundle.cjs   ← 단일 파일 ~500KB
```

## 구현 체크리스트

### FR-1-001: esbuild devDependency 추가

**파일**: `package.json`
**변경 내용**:
```json
"devDependencies": {
  "esbuild": "^0.25.0",
  // ... 기존 항목
}
```

- [ ] `npm install --save-dev esbuild` 실행
- [ ] `package-lock.json` 업데이트 확인

### FR-1-002: zod 직접 의존성 선언

**파일**: `package.json`
**변경 내용**:
```json
"dependencies": {
  "zod": "^3.25.0",
  // ... 기존 항목
}
```

- [ ] 현재 설치된 zod 버전 확인 (`npm ls zod`)
- [ ] `dependencies`에 호환 범위로 추가
- [ ] `npm install` 후 lock 파일 업데이트

### FR-1-003: 번들 빌드 스크립트 추가

**파일**: `package.json`
**변경 내용**:
```json
"scripts": {
  "bundle:mcp": "esbuild src/main/mcp-server.mjs --bundle --platform=node --target=node20 --format=cjs --outfile=src/main/mcp-server.bundle.cjs --banner:js=\"#!/usr/bin/env node\"",
  "prebuild": "npm run bundle:mcp",
  "prebuild:win": "npm run bundle:mcp",
  "prebuild:mac": "npm run bundle:mcp",
  "prebuild:linux": "npm run bundle:mcp",
  // ... 기존 항목
}
```

- [ ] `bundle:mcp` 스크립트 추가
- [ ] `prebuild*` 훅 추가 (electron-builder 실행 전 자동 번들링)
- [ ] shebang 줄이 번들 최상단에 삽입되는지 확인 (이슈 #14 해결)
- [ ] `npm run bundle:mcp` 실행 후 `src/main/mcp-server.bundle.cjs` 생성 확인

### FR-1-004: electron-builder.yml asarUnpack 축소

**파일**: `electron-builder.yml`
**변경 내용**:
```yaml
asarUnpack:
  - "src/main/mcp-server.bundle.cjs"
```

- [ ] `node_modules/**` 제거
- [ ] `src/main/frontmatter.js` 제거 (번들에 인라인됨)
- [ ] `src/main/mcp-server.mjs` → `src/main/mcp-server.bundle.cjs`로 변경

### FR-1-005: .gitignore에 번들 파일 추가

**파일**: `.gitignore`
**변경 내용**:
```
# esbuild bundle output
src/main/mcp-server.bundle.cjs
```

- [ ] 번들 산출물이 git 추적에서 제외되는지 확인

### FR-1-006: 원본 mcp-server.mjs에 shebang 추가

**파일**: `src/main/mcp-server.mjs` (line 1)
**변경 내용**: 파일 최상단에 추가
```javascript
#!/usr/bin/env node
```

- [ ] 개발 환경에서 `node src/main/mcp-server.mjs` 정상 작동 확인
- [ ] shebang은 Node.js에서 무시되므로 기존 동작에 영향 없음

## 테스트 시나리오

### TC-1-001: 번들 빌드 성공 (정상 케이스)

**Given**: 프로젝트 루트에서 `npm install` 완료
**When**: `npm run bundle:mcp` 실행
**Then**:
- `src/main/mcp-server.bundle.cjs` 파일이 생성됨
- 파일 크기 < 2MB
- 파일 첫 줄이 `#!/usr/bin/env node`

### TC-1-002: 번들된 서버 정상 시작 (정상 케이스)

**Given**: `mcp-server.bundle.cjs` 빌드 완료
**When**: `node src/main/mcp-server.bundle.cjs` 실행
**Then**:
- "DocuLight MCP Server running on stdio" 메시지 출력
- Electron 앱 미실행 시 자동 시작 시도
- SIGINT로 정상 종료 가능

### TC-1-003: 패키징 후 번들 경로 확인 (정상 케이스)

**Given**: `npm run build:win` 완료
**When**: `dist/win-unpacked/resources/` 디렉토리 확인
**Then**:
- `app.asar.unpacked/src/main/mcp-server.bundle.cjs` 존재
- `app.asar.unpacked/node_modules/` 디렉토리 **존재하지 않음**
- `node dist/.../mcp-server.bundle.cjs` 정상 실행

### TC-1-004: 번들 내 의존성 해결 (예외 케이스)

**Given**: `mcp-server.bundle.cjs` 빌드 완료
**When**: `node_modules` 디렉토리를 임시 이름으로 변경 후 번들 실행
**Then**:
- 번들이 자체적으로 모든 의존성을 포함하므로 정상 실행됨
- `zod`, `@modelcontextprotocol/sdk` import 에러 없음

### TC-1-005: 인스톨러 크기 검증 (경계값)

**Given**: `npm run build:win` 완료
**When**: `dist/DocuLight-Setup-*.exe` 파일 크기 확인
**Then**:
- 인스톨러 크기 < 120MB (이전 대비 크기 감소 확인)
