## Phase 의존성 그래프

```
Phase 1 (MCP Stdio 번들링)
    │
    ├──► Phase 5 (코드 품질) ← Phase 1의 번들 경로가 README에 반영됨
    │        │
    │        └──► Phase 6 (플랫폼) ← 공유 모듈(mcp-save.js) 완성 후 적용
    │
Phase 2 (검색 엔진)        ← Phase 1과 독립, 병렬 가능
    │
Phase 3 (HTTP 보안)        ← Phase 1, 2와 독립, 병렬 가능
    │
Phase 4 (동시성/IPC)       ← Phase 1, 2, 3과 독립, 병렬 가능
```

### 병렬 실행 가능 그룹

| 그룹 | Phase | 이유 |
|------|-------|------|
| A | 1, 2, 3, 4 | 서로 다른 파일/모듈을 수정, 교차 의존성 없음 |
| B | 5 | Phase 1 완료 필요 (번들 파일명 확정 후 README 반영) |
| C | 6 | Phase 5 완료 필요 (mcp-save.js 추출 후 file-association 관련 리팩토링) |

## 실패 시 대응 정책

### 심각도별 대응

| 심각도 | 실패 시 조치 |
|--------|------------|
| CRITICAL | 릴리즈 차단. 해당 Phase 재작업 후 전체 재검증 |
| HIGH | 릴리즈 차단. 해당 FR 재작업 후 Phase 재검증 |
| MEDIUM | 다음 릴리즈(v0.11.1)로 연기 가능. 릴리즈 노트에 Known Issue 기재 |
| LOW | 다음 릴리즈로 연기. 백로그 등록 |

### 롤백 절차

- 각 Phase 시작 전 `git tag pre-phase-N` 생성 권장
- Phase 검증 실패 시 `git revert` 또는 `git reset --hard pre-phase-N`으로 원복
- 병렬 Phase(2, 3, 4)는 독립 브랜치에서 작업 후 merge 권장

### Phase 간 실패 전파

| 실패 Phase | 영향 | 대응 |
|-----------|------|------|
| Phase 1 실패 | Phase 5, 6 대기 | Phase 2, 3, 4는 영향 없이 계속 진행 |
| Phase 2 실패 | 없음 | 다른 Phase 영향 없음 |
| Phase 3 실패 | 없음 | 다른 Phase 영향 없음 |
| Phase 4 실패 | 없음 | 다른 Phase 영향 없음 |
| Phase 5 실패 | Phase 6 대기 | Phase 1~4 영향 없음 |
| Phase 6 실패 | 없음 | 릴리즈 가능 여부는 심각도별 정책에 따름 |

### 작업 전 체크포인트 (모든 Phase 공통)

```bash
# Phase 시작 전 반드시 실행
git tag pre-phase-N        # 롤백 기점 생성
npm run build:win          # 현재 빌드 정상 확인
npx playwright test        # 현재 테스트 통과 확인
```
