## 구현 체크리스트

### FR-6-001: Linux `.desktop` 아이콘 경로 수정

**파일**: `src/main/file-association.js`
**변경 위치**: Linux 등록 함수 내 아이콘 경로 (line ~250)

```javascript
// 변경 전
const iconPath = path.join(__dirname, '..', '..', 'assets', 'icon.png');

// 변경 후
let iconPath;
if (app.isPackaged) {
  // 패키징 빌드: extraResources로 추출된 아이콘 사용
  iconPath = path.join(process.resourcesPath, 'icon.png');
} else {
  // 개발 환경: 소스 트리의 아이콘 사용
  iconPath = path.join(__dirname, '..', '..', 'assets', 'icon.png');
}
```

- [ ] `app.isPackaged` 분기로 패키징/개발 환경 구분
- [ ] 패키징 빌드에서 `process.resourcesPath` 사용
- [ ] `.desktop` 파일의 `Icon=` 경로가 실제 파일 시스템 경로를 가리키는지 확인

### FR-6-002: electron-builder.yml extraResources 추가

**파일**: `electron-builder.yml`
**변경 위치**: 최상위 레벨에 추가

```yaml
extraResources:
  - from: "assets/icon.png"
    to: "icon.png"
```

- [ ] `extraResources` 섹션 추가
- [ ] 빌드 후 `resources/icon.png` 파일 존재 확인 (asar 외부)
- [ ] `process.resourcesPath`에서 접근 가능 확인

### FR-6-003: macOS 코드 사이닝 설정 준비

**파일**: `electron-builder.yml`
**변경 위치**: `mac` 섹션

```yaml
mac:
  target:
    - target: dmg
      arch:
        - x64
        - arm64
  icon: assets/icon.png
  category: public.app-category.developer-tools
  hardenedRuntime: true
  gatekeeperAssess: false
  entitlements: build/entitlements.mac.plist
  entitlementsInherit: build/entitlements.mac.plist
```

**파일**: `build/entitlements.mac.plist` (신규 생성)
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>com.apple.security.cs.allow-jit</key>
  <true/>
  <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
  <true/>
  <key>com.apple.security.cs.allow-dyld-environment-variables</key>
  <true/>
</dict>
</plist>
```

- [ ] `hardenedRuntime: true` 추가
- [ ] entitlements plist 파일 생성
- [ ] 실제 노타라이제이션은 Apple Developer 계정 필요 (별도 설정)
- [ ] 이 Phase에서는 설정 준비만 수행, 실제 서명은 CI/CD에서 처리

> **참고**: 실제 코드 사이닝에는 Apple Developer Certificate가 필요합니다. 이 Phase에서는 `hardenedRuntime`과 entitlements 설정만 추가하여, 인증서가 설정되면 즉시 서명할 수 있도록 준비합니다.

### FR-6-004: ARM64 빌드 타겟 추가 (선택적)

**파일**: `electron-builder.yml`

```yaml
win:
  target:
    - target: nsis
      arch:
        - x64
        - arm64        # 추가
    - target: portable
      arch:
        - x64

linux:
  target:
    - target: AppImage
      arch:
        - x64
        - arm64        # 추가
    - target: deb
      arch:
        - x64
        - arm64        # 추가
  artifactName: DocuLight-${version}-${arch}.${ext}   # arch 포함으로 변경
```

- [ ] Windows NSIS에 `arm64` 추가
- [ ] Linux AppImage + deb에 `arm64` 추가
- [ ] Linux `artifactName`에 `${arch}` 포함 (파일명 충돌 방지)
- [ ] Portable 빌드는 x64 전용 유지 (ARM64 Windows는 x64 에뮬레이션 가능)

> **참고**: ARM64 빌드는 CI/CD 환경에서 크로스 컴파일 또는 ARM64 러너가 필요할 수 있습니다. GitHub Actions에서 지원 여부를 확인하세요.

## 테스트 시나리오

### TC-6-001: Linux deb 설치 후 아이콘 표시 (정상 케이스)

**Given**: Linux deb 패키지로 DocuLight 설치
**When**: 설정 → 파일 연결 등록
**Then**:
- `.desktop` 파일의 `Icon=` 경로가 `/opt/DocuLight/resources/icon.png`
- 해당 경로에 실제 파일 존재
- 데스크탑 환경에서 아이콘 정상 표시

### TC-6-002: Linux AppImage에서 아이콘 경로 (정상 케이스)

**Given**: Linux AppImage로 DocuLight 실행
**When**: 파일 연결 등록
**Then**:
- `process.resourcesPath`가 AppImage 마운트 경로 내부를 가리킴
- 아이콘 경로가 유효
- AppImage 재실행 후에도 아이콘이 표시됨 (AppImage 경로 변경 시 재등록 필요할 수 있음)

### TC-6-003: macOS hardenedRuntime 빌드 (예외 케이스)

**Given**: `hardenedRuntime: true` 설정 추가
**When**: `npm run build:mac` 실행 (인증서 없이)
**Then**:
- 빌드 자체는 성공
- DMG 생성됨
- 서명되지 않은 상태이지만 빌드 프로세스 에러 없음

### TC-6-004: Linux artifactName 아키텍처 구분 (경계값)

**Given**: ARM64 타겟 추가, `artifactName`에 `${arch}` 포함
**When**: `npm run build:linux` 실행 (x64 환경)
**Then**:
- `DocuLight-0.11.0-x64.AppImage` 파일 생성
- `DocuLight-0.11.0-x64.deb` 파일 생성
- 기존 `DocuLight-0.11.0.AppImage` 형식과 달라짐 (릴리즈 스크립트 업데이트 필요)
