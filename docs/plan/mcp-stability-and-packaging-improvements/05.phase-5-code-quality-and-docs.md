## 테스트 시나리오

### TC-5-001: 공유 모듈 저장 동작 (정상 케이스)

**Given**: `mcpAutoSave` 활성화, `mcpAutoSavePath` 설정됨
**When**: `open_markdown`으로 문서 열기
**Then**:
- `mcp-save.js`의 `saveMcpFile()`이 호출됨
- `{mcpAutoSavePath}/{YYYY-MM-DD}/{timestamp}_{title}.md` 경로에 파일 생성
- 기존 동작과 동일한 결과

### TC-5-002: mcp-http.mjs에서 공유 모듈 사용 (정상 케이스)

**Given**: HTTP MCP 서버 실행 중, `mcpAutoSave` 활성화
**When**: HTTP MCP로 `open_markdown` 호출
**Then**:
- `mcp-save.js`의 `saveMcpFile()`이 호출됨
- 저장 결과가 이전과 동일

### TC-5-003: noSave 파라미터 (예외 케이스)

**Given**: `mcpAutoSave` 활성화
**When**: `open_markdown(noSave: true)` 호출
**Then**:
- `saveMcpFile()`이 즉시 null 반환
- 파일이 저장되지 않음

### TC-5-004: AppImage 사용자 HTTP 연결 (예외 케이스)

**Given**: Linux AppImage로 DocuLight 실행
**When**: `claude mcp add --transport http doclight http://localhost:52580/mcp` 실행
**Then**:
- HTTP MCP로 정상 연결
- stdio 경로를 사용하지 않고도 MCP 기능 사용 가능

### TC-5-005: Dev 환경 auto-launch 20초 대기 (경계값)

**Given**: Electron 앱 미실행, 개발 환경
**When**: `npm run mcp` 실행 → auto-launch 시도
**Then**:
- `npx electron .`이 시작되는 동안 최대 10초 대기
- 연결 성공 후 MCP 서버 정상 작동
