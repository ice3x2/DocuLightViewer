## 1. 시스템 컨텍스트

```
┌─────────────────────────────────────────────────────────────────────┐
│                        사용자 환경                                    │
│                                                                     │

│  │ Claude       │◄────────────►│ mcp-server.mjs       │             │

│  │ Claude Code  │◄────────────►│ DocuLight Electron    │            │
│  │ / curl       │  JSON-RPC    │ Main Process          │            │
│  └──────────────┘  :52580/mcp  │  ┌─────────────────┐  │            │

│  ┌──────────────┐              │  │ index.js        │  │            │

│  │ (보안 위험)   │   위험!      │  │ search-engine   │  │            │

│                                └───────────────────────┘            │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.1 두 가지 MCP 전송 경로

| 경로 | 프로세스 | 전송 | 호스트 |
|------|---------|------|--------|
| **stdio** | `mcp-server.mjs` (독립 Node.js) | stdin/stdout + IPC 소켓 | Electron 외부 |
| **HTTP** | `mcp-http.mjs` (Electron 내장) | HTTP JSON-RPC :52580 | Electron 내부 |

**핵심 차이**: stdio 서버는 asar 외부에서 별도 Node.js 프로세스로 실행되므로 패키징 시 `asarUnpack` 필수. HTTP 서버는 Electron 내부에서 `import()`로 로딩되므로 asar 내부 OK.

## 4. 데이터 흐름

### 4.1 MCP Tool Call 흐름 (stdio)

```
Claude Desktop
    │ stdin: JSON-RPC
    ▼
mcp-server.mjs
    │ 1. Zod 스키마 검증
    │ 2. sendIpcRequest()
    │ 3. ndjson over Named Pipe
    ▼
index.js (IPC Server)
    │ 4. handleIpcMessage()
    │ 5. windowManager.createWindow() / searchEngine.search()
    │ 6. saveMcpFile() [open_markdown만!]
    │ 7. sendResponse()
    ▼
mcp-server.mjs
    │ 8. onIpcData() → pendingRequests resolve
    │ 9. MCP SDK → stdout 응답
    ▼
Claude Desktop
```

### 4.2 자동 저장 + 검색 인덱스 흐름

```
open_markdown 호출
    │
    ├─► saveMcpFile() → {mcpAutoSavePath}/{date}/{file}.md
    │
    ├─► searchEngine.markDirty() → dirty = true
    │
    └─► (나중에) search_documents 호출
            │
            ├─► ensureFresh()
            │       │
            │       └─► dirty? → rebuild()
            │               │
            │               ├─► _scanMarkdownFiles()
            │               ├─► _indexDocument() × N
            │               ├─► engine.consolidate()
            │               └─► _saveIndex()
            │
            └─► engine.search() → 결과 반환

⚠ update_markdown은 saveMcpFile()을 호출하지 않음!
⚠ ensureFresh()에 mutex가 없어 동시 rebuild 가능!
```

## 5. 패키지 구조 (현재 vs 목표)

### 5.1 현재 구조 (문제점 표시)

```
resources/
├── app.asar                          ← Electron이 읽는 아카이브
│   ├── src/main/*.js, *.mjs
│   ├── src/renderer/**
│   ├── assets/**
│   ├── node_modules/**               ← 전체 포함
│   └── package.json
│
└── app.asar.unpacked/                ← 외부 접근용
    ├── src/main/mcp-server.mjs       ✓
    ├── src/main/frontmatter.js       ✓
    └── node_modules/**               ✗ 전체 복사 (수백MB 중복!)
```

### 5.2 목표 구조 (esbuild 번들링 후)

```
resources/
├── app.asar
│   ├── src/main/*.js, *.mjs
│   ├── src/renderer/**
│   ├── assets/**
│   ├── node_modules/**
│   └── package.json
│
└── app.asar.unpacked/
    └── src/main/
        └── mcp-server.bundle.cjs     ← 단일 파일 (~500KB)
                                         모든 의존성 인라인 포함
                                         node_modules 불필요!
```
