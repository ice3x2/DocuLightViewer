### FR-3-002: SSE 동시 연결 수 제한

**파일**: `src/main/mcp-http.mjs`
**변경 위치**: GET /mcp 핸들러 (line ~496-511)

```javascript
const MAX_SSE_CONNECTIONS = 5;
let sseConnectionCount = 0;

if (sseConnectionCount >= MAX_SSE_CONNECTIONS) {
  res.writeHead(503, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({
    jsonrpc: '2.0',
    error: { code: -32603, message: 'Too many SSE connections' },
    id: null

  return;

req.on('close', () => { sseConnectionCount--; });
```

### FR-3-003: SSE keep-alive 서버 종료 시 정리

**파일**: `src/main/mcp-http.mjs`
**변경 위치**: startMcpHttpServer() 반환 객체

```javascript
// SSE 연결 추적 Set 추가
const activeSSEConnections = new Set();

// GET /mcp에서 연결 추가
activeSSEConnections.add(res);
req.on('close', () => { activeSSEConnections.delete(res); });

// 서버 종료 시 모든 SSE 연결 정리
return {
  server: httpServer,
  port: boundPort,
  close: () => {
    for (const res of activeSSEConnections) {
      res.end();
    }
    activeSSEConnections.clear();
    httpServer.close();
  }
};
```

- [ ] `activeSSEConnections` Set으로 활성 연결 추적
- [ ] `close()` 메서드에서 모든 SSE 연결 종료
- [ ] `index.js`의 `mcpHttpServer.close()` 호출 시 정상 정리 확인

### FR-3-004: HTTP body 읽기 타임아웃 (Slow Loris 방어)

**파일**: `src/main/mcp-http.mjs`
**변경 위치**: readBody() (line 391-403)

```javascript
function readBody(req, limit = 10 * 1024 * 1024) {
  return new Promise((resolve, reject) => {
    const BODY_TIMEOUT = 30_000;  // 30초
    const timer = setTimeout(() => {
      req.destroy();
      reject(new Error('Request body timeout'));
    }, BODY_TIMEOUT);

    const chunks = [];
    let size = 0;
    req.on('data', (chunk) => {
      size += chunk.length;
      if (size > limit) { clearTimeout(timer); req.destroy(); reject(new Error('Body too large')); return; }
      chunks.push(chunk);
    });
    req.on('end', () => { clearTimeout(timer); resolve(Buffer.concat(chunks).toString('utf-8')); });
    req.on('error', (err) => { clearTimeout(timer); reject(err); });
  });
}
```

- [ ] 30초 타임아웃 추가
- [ ] 타임아웃 시 `req.destroy()` 후 reject
- [ ] 정상 완료/에러 시 타이머 정리

## 테스트 시나리오

### TC-3-001: 브라우저 요청 차단 (정상 케이스)

**Given**: DocuLight HTTP MCP 서버 실행 중
**When**: `Origin: http://evil.com` 헤더를 포함한 POST 요청 전송
**Then**:
- 403 응답 반환
- JSON-RPC 에러 형식: `{ "error": { "code": -32600, "message": "Browser requests are not allowed" } }`

### TC-3-002: CLI 클라이언트 정상 작동 (정상 케이스)

**Given**: DocuLight HTTP MCP 서버 실행 중
**When**: `Origin` 헤더 없이 curl로 POST 요청 전송
**Then**:
- 정상 200 응답
- MCP tool 결과 반환

### TC-3-003: SSE 연결 제한 초과 (예외 케이스)

**Given**: 이미 5개의 SSE 연결이 열려 있음
**When**: 6번째 SSE 연결 시도 (GET /mcp)
**Then**:
- 503 응답 반환
- 기존 5개 연결은 유지됨
- 하나가 종료되면 새 연결 허용

### TC-3-004: Slow 요청 타임아웃 (예외 케이스)

**Given**: HTTP MCP 서버 실행 중
**When**: 연결을 열고 30초 이상 body 데이터를 보내지 않음
**Then**:
- 30초 후 연결 강제 종료
- 서버 리소스 정리됨

### TC-3-005: 서버 종료 시 SSE 정리 (경계값)

**Given**: 3개의 활성 SSE 연결이 있는 상태
**When**: Electron 앱 종료 (tray → Quit)
**Then**:
- 모든 SSE 연결이 정상 종료됨
- keep-alive interval 타이머 누수 없음
