# DocuLight v0.10.7 Release Notes

## New Features

- **MCP port availability check** — Settings UI now shows real-time port availability status (available/unavailable) with color indicators. Validates port range (1–65535) and empty input.
- **Default MCP port changed to 32580** — Moved from 52580 to avoid Hyper-V dynamic port reservation range (49152–65535), preventing `EACCES` binding failures on Windows.
- **MCP server bundling for packaged apps** — `mcp-server.mjs` is now bundled via esbuild into a standalone `mcp-server.bundle.mjs`, enabling stdio MCP in installed apps (NSIS, DMG, deb).
- **Windows ARM64 support** — NSIS installer now builds for ARM64 architecture.
- **Linux ARM64 + per-arch artifact naming** — AppImage and deb packages now build for ARM64 with architecture in the filename.

## Improvements

- **MCP HTTP server stability** — Added SSE connection limit (max 5), request body timeout (30s), browser origin blocking, atomic port file writes, and EACCES/EPERM retry during port discovery.
- **MCP stdio server resilience** — Deduplicated concurrent connection attempts, added write buffer full retry, increased max connection retries (10 to 20).
- **Named window race guard** — Prevents duplicate window creation when concurrent `open_markdown` calls target the same `windowName`.
- **Search engine improvements** — Batch file indexing (20 files at a time), concurrent rebuild deduplication, directory scan depth limit (10), graceful index load failure recovery.
- **Auto-save on update** — `update_markdown` now auto-saves content changes to existing saved files.
- **Auto-save module refactoring** — Extracted shared `mcp-save.js` module, eliminating code duplication between `index.js` and `mcp-http.mjs`.
- **macOS tray icon sizing** — Resized tray icon to 16x16 on macOS to prevent oversized display.
- **macOS code signing** — Added hardened runtime and entitlements for proper macOS distribution.
- **Linux icon path fix** — Packaged apps now resolve icon from `process.resourcesPath` instead of relative path.

# DocuLight v0.10.7 릴리즈 노트

## 새로운 기능

- **MCP 포트 가용성 실시간 표시** — 설정 UI에서 포트 입력 시 사용 가능/불가 상태를 색상(초록/빨강)으로 즉시 표시. 범위(1–65535) 및 빈 값 검증 포함.
- **기본 MCP 포트 32580으로 변경** — Hyper-V 동적 포트 예약 범위(49152–65535)를 피해 Windows에서 `EACCES` 바인딩 실패 방지.
- **패키징 앱용 MCP 서버 번들링** — esbuild로 `mcp-server.bundle.mjs` 단일 번들 생성. 설치된 앱(NSIS, DMG, deb)에서도 stdio MCP 사용 가능.
- **Windows ARM64 지원** — NSIS 인스톨러에 ARM64 아키텍처 빌드 추가.
- **Linux ARM64 + 아키텍처별 파일명** — AppImage, deb 패키지에 ARM64 빌드 및 아키텍처 구분 파일명 적용.

## 개선 사항

- **MCP HTTP 서버 안정성** — SSE 연결 수 제한(최대 5), 요청 본문 타임아웃(30초), 브라우저 Origin 차단, 포트 파일 원자적 쓰기, 포트 탐색 시 EACCES/EPERM 재시도.
- **MCP stdio 서버 복원력** — 동시 연결 시도 중복 제거, 쓰기 버퍼 풀 재시도, 최대 재연결 횟수 증가(10→20).
- **Named window 경합 방지** — 동일 `windowName`에 대한 동시 `open_markdown` 호출 시 중복 창 생성 방지.
- **검색 엔진 개선** — 배치 파일 인덱싱(20개씩), 동시 리빌드 중복 제거, 디렉토리 스캔 깊이 제한(10), 인덱스 로드 실패 시 자동 복구.
- **업데이트 시 자동 저장** — `update_markdown`이 기존 저장 파일에 변경 내용을 자동 반영.
- **자동 저장 모듈 리팩토링** — 공유 `mcp-save.js` 모듈로 추출하여 `index.js`와 `mcp-http.mjs` 간 코드 중복 제거.
- **macOS 트레이 아이콘 크기 수정** — macOS에서 트레이 아이콘을 16x16으로 리사이즈하여 과대 표시 문제 해결.
- **macOS 코드 서명** — hardened runtime 및 entitlements 추가로 macOS 배포 지원.
- **Linux 아이콘 경로 수정** — 패키징된 앱에서 `process.resourcesPath`로 아이콘 경로 해석.
